# Experiments

Record product, usability, accessibility, or device-compatibility experiments
here when their evidence should influence future work.

Each experiment should record:

- Baseline and the exact one-variable change where possible.
- Hypothesis and user/platform scope.
- Evidence, including source notes and test/device details.
- Metrics or observable acceptance criteria.
- Approval, rejection, rollback, or promotion status.

Experiments must never contain exported user data or unredacted personal/device
diagnostics.
