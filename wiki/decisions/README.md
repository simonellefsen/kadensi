# Decisions

Architecture and workflow decision records belong here. Add a short,
lowercase-kebab-case file for any material choice that future agents should not
need to rediscover.

Each decision should record:

- Context and source links.
- Decision and status.
- Consequences and compatibility impact.
- Alternatives considered.
- Links to code, tests, device evidence, and any follow-up roadmap item.

Examples needing a decision record include changes to local-only data handling,
programme/session-ID migrations, PWA update behaviour, or any backend/telemetry
proposal.
